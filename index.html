Firebase / Google 登入設定

把示範資料改成雲端同步

本頁已保留 Firebase 串接點，將下方設定填入程式碼即可啟用。

  1. 到 Firebase Console 建立專案，新增 Web App，複製 firebaseConfig。
  2. Authentication → Sign-in method 啟用 Google，Authorized domains 加入你的網域。
  3. Firestore Database 建立資料庫，新增 events collection。
  4. 在本檔案搜尋 firebaseConfig，填入設定並取消 FIREBASE_ENABLED。
  5. 部署前先把 Firestore Rules 調整為「登入者可讀寫活動」，正式上線再加強主辦權限。
  6. LINE 通知可以串：建議用 Cloud Functions 呼叫 LINE Messaging API 或 LINE Notify 類型的後端 webhook；不要把 Channel access token 放在前端。活動新增或有人加入後，由前端寫入 Firestore，再由後端觸發推播。
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /events/{eventId} {
      allow read: if true;
      allow create, update: if request.auth != null;
      allow delete: if request.auth != null && request.auth.uid == resource.data.hostUid;
    }
  }
}

LINE 推播做法：建立 LINE Developers Provider 與 Messaging API Channel，把 Channel access token 存在 Firebase Functions 環境變數；監聽 events 新增或更新後，用後端送出「新活動／有人加入／付款提醒」。前端本頁已在建立與加入活動時呼叫 sendLineNotification()，目前為示範 log，正式上線請改成呼叫你的 Cloud Function HTTPS endpoint。

exports.notifyLineOnEventWrite = onDocumentWritten('events/{eventId}', async (event) => {
  // 用 Functions Secret 讀 LINE token，組訊息後 POST 到 LINE Messaging API
});
